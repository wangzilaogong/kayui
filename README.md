# GIt规范
Type  
必须为下列之一:  
feat：新功能（feature）  
fix：修补bug  
docs：文档修改  
style： 不影响代码含义的修改(例如：white-space; 格式化等) 
refactor：重构（即不是新增功能，也不是修改bug的代码变动）  
perf: 提升性能的修改  
test：增加或修改测试  
chore：构建流程或辅助工具的变动
